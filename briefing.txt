    DATABASE
    --------
    File: database.db (located in the root project folder)
    Full path: C:\Users\PC\Desktop\imad docc projet fin d'anner\gestion-voiture\database.db
    Engine: SQLite
    Viewer tool: DB Browser for SQLite (sqlitebrowser.org) — open database.db to browse/edit data
    OR: SQLite Viewer extension in VS Code

    The app writes directly to this file. Any car/client/sale added through the website
    is saved here instantly.


    DATABASE SCHEMA (tables)
    ------------------------
    - marque          → car brands
    - modele          → car models (linked to marque)  ← MODIFIED (see below)
    - voiture         → cars (VIN, model, type, plate, year, buy price, sell price, status)
    - client          → customers
    - fournisseur     → suppliers
    - agent           → sales agents (with commission rate %)
    - commande_achat  → purchase orders (from suppliers)
    - ligne_achat     → cars per purchase order (links commande_achat → voiture by VIN)
    - vente           → sales transactions
    - ligne_vente     → cars per sale (links vente → voiture by VIN)
    - facture         → invoices  ← NOW ACTUALLY WRITTEN ON EVERY SALE
    - paiement        → payments
    - historique_proprietaire → ownership history
    - devis           → quote requests (client devis history)  ← ADDED TO init_db
    - user            → app users (roles: admin / user)

    User roles:
    admin → full access to everything
    user  → browse all brands, see available cars, request a devis (quote)

    FEATURES IMPLEMENTED
    --------------------
    The following features have been fully implemented. This list is the source of truth
    for what the app can and cannot do.

    ✅ 1. Edit & Delete cars
     Routes: /voitures/edit/<vin>  (GET/POST)
             /voitures/delete/<vin> (POST)
     Files:  templates/voiture_edit.html, app.py

    ✅ 2. Search & filter on voitures list
     Filter by: brand, model, plate, VIN (text search)
                status chips: Tous / En Stock / Réservées / Vendues
                type chips:   Tous types / Neuf / Occasion
                price range:  Min / Max MAD inputs
                year range:   De / À inputs
     Implementation: client-side JS in templates/voitures.html
     Réinitialiser button clears all filters at once.

    ✅ 3. Client "Mes Devis" page
     Route: /client/mes-devis
     Shows the logged-in client's full quote history (date, vehicle, price HT/TTC, status).
     Devis are saved to the devis table on each request.
     Files: templates/mes_devis.html, app.py

    ✅ 4. Edit & Delete clients, agents, suppliers
     Routes: /clients/edit/<id>       /clients/delete/<id>
             /agents/edit/<id>        /agents/delete/<id>
             /fournisseurs/edit/<id>  /fournisseurs/delete/<id>
     Files:  templates/client_edit.html, templates/agent_edit.html,
             templates/fournisseur_edit.html, app.py

    ✅ 5. Monthly sales chart on dashboard
     Dashboard fetches last 12 months of sales (count + revenue per month).
     Data passed to index.html as JSON (chart_labels, chart_ventes, chart_ca).
     Rendered with Chart.js canvas.
     Files: templates/index.html, app.py (dashboard route)

    ✅ 6. Car detail page
     Route: /voitures/<vin>
     Shows full car info. Admin sees Edit / Delete buttons.
     Files: templates/voiture_detail.html, app.py

    ✅ 7. Filter cars by price range, year, and type
     Price range: Min / Max MAD number inputs (filters as you type)
     Year range:  De / À number inputs (filters as you type)
     Type:        Neuf / Occasion chips
     Reset button clears all filters at once.
     Files: templates/voitures.html

    ✅ 8. My Account page for clients
     Route: /client/mon-compte  (GET/POST)
     Clients can view and update their name, phone, email.
     Files: templates/mon_compte.html, app.py

    ✅ 9. Print-ready invoice (facture) after a sale
     Route: /confirmation/<id_vente>
     Admin is now auto-redirected here after recording a sale.
     facture row is now inserted into the DB on every sale.
     Files: templates/confirmation.html, app.py

    ✅ 10. Flash messages (success/error feedback)
     All add/edit/delete actions now show a coloured confirmation message.
     Displayed in templates/base.html using Flask flash().
     Covers: voitures, clients, agents, fournisseurs, ventes.

    ✅ 11. Cancel a sale (vente_delete)
     Route: /ventes/delete/<id_vente> (POST, admin only)
     Cancels the sale, cascades deletes (paiement→facture→ligne_vente→vente),
     and restores the car's status to en_stock automatically.
     Files: app.py, templates/transactions.html

    ✅ 12. Marge Brute on dashboard
     Shows total gross profit (sum of prix_vente - prix_achat) across all sales.
     Displayed as a stat card alongside Chiffre d'affaires.
     Files: templates/index.html, app.py (dashboard route)

    ✅ 13. Clickable VINs in Ventes & Achats
     VIN codes in the transactions page now link to the car detail page.
     Files: templates/transactions.html

    ❌ Password hashing — NOT implemented (out of scope for local project).
     Passwords are stored and compared as plain text. Acceptable for a local school project.
