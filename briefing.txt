DATABASE SCHEMA (all tables)
-----------------------------
Original tables (from initial project):
  - marque               -> car brands
  - modele               -> car models (linked to marque)
  - voiture              -> cars (VIN, model, type, plate, year, buy/sell price, status)
  - client               -> customers
  - fournisseur          -> suppliers
  - agent                -> sales agents (with commission rate %)
  - commande_achat       -> purchase orders (from suppliers)
  - ligne_achat          -> cars per purchase order (commande_achat <-> voiture)
  - vente                -> sales transactions
  - ligne_vente          -> cars per sale (vente <-> voiture)
  - facture              -> invoices (linked to vente, NOW written on every sale)
  - paiement             -> payments (linked to facture)
  - historique_proprietaire -> ownership history
  - user                 -> app users (roles: admin / user)

Added by agent:
  - devis                -> quote requests made by clients (NEW TABLE — see MCD/MLD below)

User roles:
  admin -> full access to everything
  user  -> browse brands, see available cars, request a devis (quote)

CHANGE B — NEW TABLE: devis
-----------------------------
Table:   devis
Purpose: Records every quote (devis) a client requests for a specific car.
         Clients see their history on the "Mes Devis" page.
         Admins can also generate quotes via the Devis menu.

Full schema:
  devis(
    id_devis   INTEGER  PRIMARY KEY AUTOINCREMENT,
    vin        TEXT     NOT NULL,    -- references voiture.vin
    id_client  INTEGER  NOT NULL,    -- references client.id_client
    date_devis DATE,
    prix_vente REAL,                 -- car sell price at time of quote
    devis_num  TEXT                  -- readable quote number, e.g. 20260718-XXXX
  )
  Foreign keys:
    vin        -> voiture(vin)
    id_client  -> client(id_client)

How added: table existed in live database.db before Change 5;
           formally added to init_db() in Change 5 (safe on fresh installs now)

MCD impact:
  NEW ENTITY: Devis
    Attributes: id_devis (PK), date_devis, prix_vente, devis_num

  NEW ASSOCIATIONS:
    Client  (1,1) ---< demande  >--- (0,n) Devis
    Voiture (1,1) ---< concerne >--- (0,n) Devis

  Reading:
    - A Client can make zero or many Devis (0,n)
    - A Devis belongs to exactly one Client (1,1)
    - A Voiture can be referenced in zero or many Devis (0,n)
    - A Devis concerns exactly one Voiture (1,1)

MLD impact:
  NEW RELATION:
    devis(id_devis, date_devis, prix_vente, devis_num, vin#, id_client#)

  Foreign keys:
    vin#       -> voiture(vin)
    id_client# -> client(id_client)

FEATURES IMPLEMENTED
--------------------
This is the authoritative list of what the app can and cannot do.

[OK] 1. Edit & Delete cars
   Routes: /voitures/edit/<vin>  (GET/POST), /voitures/delete/<vin> (POST)
   Files:  templates/voiture_edit.html, app.py

[OK] 2. Search & filter on voitures list
   - Text search: brand, model, plate, VIN
   - Status chips: Tous / En Stock / Reservees / Vendues
   - Type chips: Tous types / Neuf / Occasion
   - Price range: Min / Max MAD (live, as you type)
   - Year range: De / A (live, as you type)
   - Reset button clears all filters at once
   100% client-side JS — no backend or DB changes needed
   File: templates/voitures.html

[OK] 3. Client "Mes Devis" page
   Route: /client/mes-devis
   Shows client's full quote history. Each request saves a row to the devis table.
   Files: templates/mes_devis.html, app.py

[OK] 4. Edit & Delete clients, agents, suppliers
   Routes: /clients/edit/<id>, /clients/delete/<id>
           /agents/edit/<id>, /agents/delete/<id>
           /fournisseurs/edit/<id>, /fournisseurs/delete/<id>
   Files: templates/client_edit.html, agent_edit.html, fournisseur_edit.html, app.py

[OK] 5. Monthly sales chart on dashboard
   Last 12 months fetched and passed as JSON to Chart.js (count + revenue per month).
   Files: templates/index.html, app.py

[OK] 6. Car detail page
   Route: /voitures/<vin> — full car info, Edit/Delete for admin.
   Files: templates/voiture_detail.html, app.py

[OK] 7. Filters by price range, year, type — see Feature 2.

[OK] 8. My Account page for clients
   Route: /client/mon-compte (GET/POST) — view and update name, phone, email.
   Files: templates/mon_compte.html, app.py

[OK] 9. Invoice (facture) after sale
   Route: /confirmation/<id_vente>
   Admin is auto-redirected here after recording a sale.
   A facture row is inserted into the DB on every sale (was broken before Change 5).
   Files: templates/confirmation.html, app.py

[OK] 10. Flash messages — feedback on every action
    Coloured success/error banners after every add/edit/delete action.
    Files: templates/base.html (display block), app.py (flash() calls on all routes)

[OK] 11. Cancel a sale (vente_delete)
    Route: /ventes/delete/<id_vente> (POST, admin only)
    Cascades: paiement -> facture -> ligne_vente -> vente, then car status -> en_stock.
    Files: app.py, templates/transactions.html

[OK] 12. Marge Brute on dashboard
    Gross profit card: sum of (prix_vente - prix_achat) across all completed sales.
    Files: templates/index.html, app.py

[OK] 13. Clickable VINs in Ventes & Achats
    VIN codes in the transactions page link directly to the car detail page.
    File: templates/transactions.html

[NO] Password hashing — NOT implemented. Out of scope for local school project.
     Passwords stored and compared as plain text (acceptable locally).
